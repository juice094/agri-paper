# 中文文献视角下的研究空白（Gap Analysis）

基于对《农业工程学报》《农业图书情报学报》《中国农机化学报》《智慧农业(中英文)》等中文核心期刊（北大核心/CSCD/AMI核心，2019–2025）的系统检索，共梳理出 15 篇高度相关文献。以下从中文文献视角补充 **2 个关键研究空白（Gap）**，可与已有的英文文献矩阵形成互补。

---

## Gap 1：农业信息系统的"配置驱动/声明式架构"研究在中文核心文献中几乎空白

### 观察
中文核心文献对农业信息化的研究高度集中于**知识内容层**（知识图谱构建、专家系统规则库、农业文本挖掘、智能问答），而在**系统架构层**关于"配置驱动""声明式系统""模型驱动工程（MDE）""DSL（领域特定语言）"等软件工程范式的研究几乎不存在。

- 检索关键词组合："配置驱动+农业""声明式+农业信息系统""DSL/模型驱动+农业""本地优先+农业知识管理"在知网、万方、维普、百度学术中均未检索到 2019–2025 年间的中文核心期刊文献。
- 现有农业专家系统/知识服务平台的研究（如赵瑞雪等 2020；陈朗等 2023；赵春江 2023）多采用**传统的三层架构或定制化编码实现**，系统功能与代码紧耦合，扩展性和可移植性受限。

### 启示
这意味着：
1. 国内农业信息系统研究仍停留在"为每个场景单独开发系统"的作坊模式，缺乏**通过配置即可适配不同作物、不同区域、不同业务场景**的通用架构探索。
2. 本研究提出的"配置驱动的本地优先农业Agent系统"（基于 Rust Clarity Agent 框架 + devbase 知识管理 + MCP 协议）在中文语境下具有显著的**架构创新性**，可以填补上述空白。

### 可引用策略
- 引用穆维松等（2023）、赵春江（2023）说明农业知识图谱与智能服务的技术成熟度已较高；
- 转折指出：现有中文文献尚未解决"知识服务系统如何以声明式配置快速适配多样化农业场景"的问题，从而引出本研究的配置驱动架构设计。

---

## Gap 2：农业知识服务中的"智能体协同与MCP协议化服务封装"尚未进入中文核心研究视野

### 观察
2024–2025 年，《农业图书情报学报》等期刊开始关注 Agent 在图书情报领域的应用（如钱力等 2025；刘炜等 2025），但存在两个明显局限：

1. **领域错位**：现有中文 Agent 文献集中在图书馆服务、科研情报、科技文献管理（如 DIS Agent、A-LSP 架构），**尚未延伸至农业生产管理一线场景**（如田间病虫害诊断、精准施肥决策、农机调度）。
2. **协议层缺失**：刘炜等（2025）虽然提出了基于 MCP（Model Context Protocol）的 API 封装思路，但属于**图书馆服务平台（LSP）**的跨领域借鉴，**农业领域尚无 MCP 协议化知识服务的研究报道**。

### 启示
- 中文文献证实了 Agent + MCP 在知识服务领域的前瞻性（刘炜等 2025），但农业场景的 Agent 化改造、农业知识工具的 MCP 封装、多 Agent 协同决策等仍是一片蓝海。
- 本研究将 MCP 协议与农业知识管理（devbase）相结合，实现农业 Agent 的**工具发现-调用-协同**闭环，在国内研究中属于**率先探索**。

### 可引用策略
- 引用钱力等（2025）、刘炜等（2025）证明 Agent 与 MCP 已成为国内图情领域的前沿方向；
- 指出农业领域对 Agent 协同与 MCP 服务封装的研究尚处空白，进而论证本研究"农业 Agent + MCP 协议 + 本地优先知识管理"三位一体的创新价值。

## 总结：中英文文献的互补定位

| 维度 | 英文文献（W1 矩阵） | 中文文献（W3 矩阵） |
|------|-------------------|-------------------|
| 时间跨度 | 2021–2026 | 2019–2025 |
| 核心优势 | 本地优先软件、Agent 架构、LLM 工具使用（MCP/Grounding）理论前沿 | 农业知识图谱、专家系统、农业信息化与知识服务工程实践 |
| 主要空白 | 农业领域适配不足、本土化农业知识管理案例较少 | 配置驱动/声明式架构空白；农业 Agent/MCP 协同研究缺失 |
| 本研究的衔接价值 | 引入国际前沿的 Agent 与本地优先架构理论 | 对接国内成熟的农业知识图谱与专家系统成果，填补架构层空白 |

> **一句话结论**：中文核心文献为"农业知识内容智能化"提供了坚实的本土化工程支撑，但尚未触及"农业信息系统架构现代化"议题——这正是本研究"配置驱动的本地优先农业Agent系统"的独特切入点。
